## defect inventory

### defect 1: PR title type not allowed

**what:** pullreq-title check failed: `Unknown release type "cont" found in pull request title`

**how:** commit message used `cont(practs)` type, which is allowed by commitlint but not by the PR title checker action (`amannn/action-semantic-pull-request@v5`).

```
# allowed by PR title checker
fix, feat, chore

# used in commit
cont(practs)
```

**why:** the PR title is auto-set from the first commit message header. the PR title checker is more restrictive than commitlint, only accept `fix`, `feat`, `chore`.

**fix:** human must update PR title to use `fix` instead of `cont`:
```sh
gh pr edit 48 --title "fix(practs): repair test defects from declapract upgrade"
```

**status:** wait human action

### defect 2: ESM module fix gitignored

**what:** unit tests fail in CI with `Cannot use import statement outside a module` but pass locally

**how:** the fix for ESM module load (nested package.json with `"type": "module"`) was created locally but never committed because the exampleProject's `.gitignore` contains `provision/schema/declarations/**/*` which blocks all files in that directory.

```
# .gitignore pattern
provision/schema/declarations/**/*

# blocked file
provision/schema/declarations/package.json
```

**why:** the gitignore pattern is overly broad. it intends to ignore generated schema files but also blocks infrastructure files needed for module resolution.

**fix:** add negation pattern to allow package.json while other files remain ignored:
```sh
# added to .gitignore
!provision/schema/declarations/package.json
```

**status:** fixed

### defect 3: test database npm command name mismatch

**what:** integration and acceptance tests fail with `AggregateError` because database connection is undefined

**how:** declapract upgrade changed workflow to call `npm run start:testdb --if-present` but package.json only has `provision:integration-test-db`. since `--if-present` silently skips, the database never starts.

```yaml
# workflow expects:
- name: start:testdb
  run: npm run start:testdb --if-present

# package.json has:
"provision:integration-test-db": "npm run provision:docker:clear && ..."
```

**why:** declapract practice uses `start:testdb` convention. this repo used `provision:integration-test-db` which is valid but not matched by the new workflow.

**fix:** add alias in package.json:
```json
"start:testdb": "npm run provision:integration-test-db"
```

**status:** fixed

### defect 4: test:integration provision step removed

**what:** integration tests fail with `ENOENT: no such file or directory, open '.../provision/schema/sql/domain.yml'`

**how:** declapract upgrade replaced test:integration command but lost the provision step. the old command was:
```
"test:integration": "npm run test:integration:provision-generated-files-to-import && jest ..."
```
the new command was:
```
"test:integration": "set -eu && jest ..."
```

**why:** the provision step runs `generate.test.ts` which generates SQL files that integration tests depend on. without it, files don't exist in CI.

**fix:** restore provision step to test:integration command:
```json
"test:integration": "set -eu && npm run test:integration:provision-generated-files-to-import && jest ..."
```

**status:** fixed

---

### defect 5: provision command path stale

**what:** provision step fails with "0 matches" because test file path is stale

**how:** declapract upgrade moved files from `src/logic/` to `src/domain.operations/` but the provision command still references the old path:
```
# package.json references:
npx jest src/logic/commands/generate.test.ts

# actual file location:
src/domain.operations/commands/generate.test.ts
```

**why:** the provision command was not updated when the directory structure changed from `logic/` to `domain.operations/`.

**fix:** update provision command path:
```json
"test:integration:provision-generated-files-to-import": "npx jest src/domain.operations/commands/generate.test.ts"
```

**status:** fixed
