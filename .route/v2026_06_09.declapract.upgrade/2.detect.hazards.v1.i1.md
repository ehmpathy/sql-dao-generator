# 2.detect.hazards — hazard detection report

## summary

identified **4 categories** of hazards from the declapract upgrade.

## 1. test hazards

### jest 30 major version bump

**what changed:**
- jest: 29.3.1 → 30.2.0
- @types/jest: 29.2.4 → 30.0.0

**why it may break:**
- jest 30 has breaking api changes
- test utilities may behave differently
- snapshot format may differ

**how to verify:**
```sh
rhx git.repo.test --what unit --mode apply
rhx git.repo.test --what integration --mode apply
```

### jest transform changed: ts-jest → @swc/jest

**what changed:**
- all jest configs now use `@swc/jest` instead of `ts-jest`
- `core-js` setup file removed

**why it may break:**
- swc may handle typescript edge cases differently
- missing polyfills from core-js removal

**how to verify:**
```sh
rhx git.repo.test --what unit --mode apply
rhx git.repo.test --what integration --mode apply
```

### acceptance directory renamed: acceptance/ → blackbox/

**what changed:**
- `acceptance/environment.ts` → `blackbox/environment.ts`
- jest.acceptance.config.ts updated to match

**why it may break:**
- imports referencing old path will fail
- ci workflows may reference old directory

**how to verify:**
```sh
rhx git.repo.test --what acceptance --against local --env test --mode apply
```

### test asset paths restructured

**what changed:**
- `src/logic/__test_assets__` → `src/domain.operations/.test.assets`
- some assets also at `src/domain.operations/__test_assets__`

**why it may break:**
- relative imports in test files may break
- snapshot paths may change

**how to verify:**
```sh
rhx git.repo.test --what unit --mode apply
rhx git.repo.test --what integration --mode apply
```
