### defect 10: pg DeprecationWarning treated as test error

**classification:** adoption candidate

**rationale:** pg@8.14 writes deprecation warnings to stderr. the "any stderr = error" pattern is fragile. the pattern to filter warnings from stderr before check for errors could benefit other repos.

**code pattern:**
```typescript
const stderrWithoutWarnings = result.stderr
  .split('\n')
  .filter((line) => !line.includes('DeprecationWarning'))
  .filter((line) => !line.includes('--trace-deprecation'))
  .join('\n')
  .trim();
if (stderrWithoutWarnings) throw new Error(stderrWithoutWarnings);
```

**action:** consider document this pattern in ehmpathy practices for shell exec error handler.
